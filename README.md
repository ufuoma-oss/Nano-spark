Your Personal AI Assistant; easy to install, deploy on your own machine or on the cloud; supports multiple chat apps with easily extensible capabilities.

> **Core capabilities:**
>
> **Every channel** — DingTalk, Feishu, QQ, Discord, iMessage, and more. One assistant, connect as you need.
>
> **Under your control** — Memory and personalization under your control. Deploy locally or in the cloud; scheduled reminders to any channel.
>
> **Skills** — Built-in cron; custom skills in your workspace, auto-loaded. No lock-in.
>
> <details>
> <summary><b>What you can do</b></summary>
>
> <br>
>
> - **Social**: daily digest of hot posts (Xiaohongshu, Zhihu, Reddit), Bilibili/YouTube summaries.
> - **Productivity**: newsletter digests to DingTalk/Feishu/QQ, contacts from email/calendar.
> - **Creative**: describe your goal, run overnight, get a draft next day.
> - **Research**: track tech/AI news, personal knowledge base.
> - **Desktop**: organize files, read/summarize docs, request files in chat.
> - **Explore**: combine Skills and cron into your own agentic app.
>
> </details>

## Quick Start

### pip install (recommended)

If you prefer managing Python yourself:

```bash
pip install nanospark
nanospark init --defaults
nanospark app
```

Then open **http://127.0.0.1:8088/** in your browser for the Console (chat with NanoSpark, configure the agent). To talk in DingTalk, Feishu, QQ, etc., add a channel in the [docs](https://nanospark.onrender.com/docs/channels).

![Console](https://img.alicdn.com/imgextra/i4/O1CN01iuGyNc1mNwsUU5NQI_!!6000000004943-2-tps-3822-2070.png)

### One-line install (beta, continuously improving)

No Python required — the installer handles everything:

**macOS / Linux:**

```bash
curl -fsSL https://nanospark.onrender.com/install.sh | bash
```

**Windows (PowerShell):**

```powershell
irm https://nanospark.onrender.com/install.ps1 | iex
```

Then open a new terminal and run:

```bash
nanospark init --defaults   # or: nanospark init (interactive)
nanospark app
```

<details>
<summary><b>Install options</b></summary>

**macOS / Linux:**

```bash
# Install a specific version
curl -fsSL ... | bash -s -- --version 0.0.2

# Install from source (dev/testing)
curl -fsSL ... | bash -s -- --from-source

# With local model support
bash install.sh --extras llamacpp    # llama.cpp (cross-platform)
bash install.sh --extras mlx         # MLX (Apple Silicon)
bash install.sh --extras llamacpp,mlx

# Upgrade — just re-run the installer
curl -fsSL ... | bash

# Uninstall
nanospark uninstall          # keeps config and data
nanospark uninstall --purge  # removes everything
```

**Windows (PowerShell):**

```powershell
# Install a specific version
irm ... | iex; .\install.ps1 -Version 0.0.2

# Install from source (dev/testing)
.\install.ps1 -FromSource

# With local model support
.\install.ps1 -Extras llamacpp      # llama.cpp (cross-platform)
.\install.ps1 -Extras mlx           # MLX
.\install.ps1 -Extras llamacpp,mlx

# Upgrade — just re-run the installer
irm ... | iex

# Uninstall
nanospark uninstall          # keeps config and data
nanospark uninstall --purge  # removes everything
```

### Using Docker

```bash
docker pull agentscope/nanospark:latest
docker run -p 8088:8088 -v nanospark-data:/app/working agentscope/nanospark:latest
```

Then open **http://127.0.0.1:8088/** for the Console. Config, memory, and skills are stored in the `nanospark-data` volume. To pass API keys (e.g. `DASHSCOPE_API_KEY`), add `-e VAR=value` or `--env-file .env` to `docker run`.

The image is built from scratch. To build the image yourself, please refer to the [Build Docker image](scripts/README.md#build-docker-image) section in `scripts/README.md`, and then push to your registry.

### Using ModelScope

**No local install?** [ModelScope Studio](https://modelscope.cn/studios/fork?target=AgentScope/NanoSpark) one-click cloud setup. Set your Studio to **non-public** so others cannot control your NanoSpark.

### Deploy on Alibaba Cloud ECS

To run NanoSpark on Alibaba Cloud (ECS), use the one-click deployment: open the [NanoSpark on Alibaba Cloud (ECS) deployment link](https://computenest.console.aliyun.com/service/instance/create/cn-hangzhou?type=user&ServiceId=service-1ed84201799f40879884) and follow the prompts. For step-by-step instructions, see [Alibaba Cloud Developer: Deploy your AI assistant in 3 minutes](https://developer.aliyun.com/article/1713682).

## API Key

If you use a **cloud LLM** (e.g. DashScope, ModelScope), you must set an API key before chatting. NanoSpark will not work until a valid key is configured.

**Where to set it:**

1. **`nanospark init`** — When you run `nanospark init`, the command has a step to configure the LLM provider and API key. Follow the prompts to choose a provider and enter your key.
2. **Console** — After `nanospark app`, open **http://127.0.0.1:8088/** → **Settings** → **Models**. Select a provider, fill in the **API Key** field, then activate that provider and model.
3. **Environment variable** — For DashScope you can set `DASHSCOPE_API_KEY` in your shell or in a `.env` file in the working directory.

Tools that need extra keys (e.g. `TAVILY_API_KEY` for web search) can be set in Console **Settings → Environment variables**, or see [Config](https://nanospark.onrender.com/docs/config) for details.

> **Using local models only?** If you use [Local Models](#local-models) (llama.cpp or MLX), you do **not** need any API key.

## Local Models

NanoSpark can run LLMs entirely on your machine — no API keys or cloud services required.

| Backend       | Best for                                 | Install                         |
| ------------- | ---------------------------------------- | ------------------------------- |
| **llama.cpp** | Cross-platform (macOS / Linux / Windows) | `pip install 'nanospark[llamacpp]'` |
| **MLX**       | Apple Silicon Macs (M1/M2/M3/M4)         | `pip install 'nanospark[mlx]'`      |

After installing, download a model and start chatting:

```bash
nanospark models download Qwen/Qwen3-4B-GGUF
nanospark models # select the downloaded model
nanospark app # start the server
```

You can also download and manage local models from the Console UI.

## Install from source

```bash
git clone https://github.com/ufuoma-oss/Nano-spark.git
cd NanoSpark
pip install -e .
```

- **Dev** (tests, formatting): `pip install -e ".[dev]"`
- **Console** (build frontend): `cd console && npm ci && npm run build`, then `nanospark app` from project root.
